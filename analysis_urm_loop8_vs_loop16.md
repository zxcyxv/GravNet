## 6. Carry Cycle 진폭 비교

Carry reset은 avg_steps가 리셋되는 시점에서 발생. Loop 16은 최대 16 steps까지 carry가 깊어지므로 리셋 시 충격이 더 큼.

| Reset 포인트 (추정) | Loop 8 스파이크 | Loop 16 스파이크 |
|---|---|---|
| ~400 | 1.27 | 1.32 |
| ~800 | 0.56 | **2.19** |
| ~1200 | 2.37 | **3.56** |
| ~1600 | 1.69 | **4.94** |
| ~2000 | 1.09 | **6.21** |
| ~2400 | 0.85 | **7.20** |

**Loop 16 carry reset 진폭이 단조 증가** (1.3 → 7.2). Loop 8에서는 이런 증가 추세 없음 (오히려 감소).

→ 깊은 carry (16 steps)에서 모델이 더 장기적인 의존성을 학습하고, reset 시 축적된 gradient가 한꺼번에 분출.

## 7. Loop 8 vs Loop 16: 학습 역학 질적 차이 요약

| 지표 | Loop 8 | Loop 16 |
|---|---|---|
| gnorm 범위 | 0.5 ~ 2.4 | 0.4 ~ **7.2** |
| lm_head 비중 | **40-65%** (독점) | **2-5%** (미미) |
| 내부 down_proj | 0.06 ~ 0.24 | **0.35 ~ 0.50** |
| 내부 qkv_proj | 0.06 ~ 0.33 | **0.11 ~ 0.64** |
| embed_tokens 추세 | 증가 (보상) | 감소 (불필요) |
| carry reset 진폭 | 안정/감소 | **단조 증가** |
| 학습 주체 | lm_head + embed | **내부 레이어** |

## 8. AMK-PD (2L→5L) vs URM (loop8→loop16) 대비

AMK-PD에서 layer 수를 2→5로 늘렸을 때 관측된 "특이점"이 URM에서 loop 수를 8→16으로 늘렸을 때도 **동일하게 재현**됨:

| 현상 | AMK-PD (2L→5L) | URM (loop8→loop16) |
|---|---|---|
| lm_head 비중 붕괴 | 68% → 15% | 62% → **2%** |
| 내부 gradient 성장 | 3-8x | 3-5x |
| gnorm 스파이크 증폭 | 있음 | **더 극적** (8.5x) |
| embed 의존도 하락 | pos_emb 유지 | embed_tokens 하락 |

### 공통 메커니즘: 계산 깊이(Computational Depth) 임계점

두 모델 모두 **총 연산 깊이** (= num_layers × loops)가 임계값을 넘으면 학습 역학이 질적으로 전환됨:

- **얕은 깊이** (AMK-PD 2L×16=32, URM 4L×8=32): lm_head 독점, 내부 레이어 과소 학습
- **깊은 깊이** (AMK-PD 5L×16=80, URM 4L×16=64): lm_head 주변화, 내부 레이어 주력화

**임계 깊이는 대략 32~64 사이**에 존재. 이 지점을 넘으면:
1. 역전파 경로가 충분히 길어져 내부 파라미터의 야코비안 체인이 유의미한 gradient를 전달
2. lm_head의 직접적 영향력이 희석되어 내부 학습이 해방
3. carry cycle에서 축적되는 장기 의존성이 reset 시 폭발적 gradient로 전환

### URM이 더 극적인 이유

URM loop16의 lm_head 비중(2%)이 AMK-PD 5L의 비중(15%)보다 훨씬 낮은 이유:
- URM은 **RMSNorm post-addition**이 원래 설계에 내장 → 분산 팽창 없이 깨끗한 역전파
- AMK-PD는 Post-Addition Norm을 사후 적용했지만, polynomial kernel mean shift의 비선형성이 추가 bottleneck으로 작용할 가능성
- URM의 softmax attention은 gradient 전파에 더 안정적
